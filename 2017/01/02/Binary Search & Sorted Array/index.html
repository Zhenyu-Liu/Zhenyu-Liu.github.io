Binary Search and Sorted Array

Posted on 2017-01-02

Classical Binary Search

Given an sorted integer array - nums, and an integer - target. Find the first/last position of target in nums, return -1 if target doesn’t exist.

Recursion or While-Loop

  • Difference: All works, but while loop better.
  • Recursion may cause trouble (avoid recursion,like Stack overflow)
  • while loop gives extra points (no infinite loop, which is harder to implement)

$\textbf{Complexity}$

Complexity for Binary search: $\Theta = log_2 n$
In java complexity for the Array.sort(Quick sort): $\Theta = nlog_2 n$

$\textbf{When to use binary search}$

  • Find first/last of something
  • $\Theta (n)$ is not available, need $\Theta (log n)$ solution.
  • Normal algorithm only binary search have $\Theta (log n)$.

Search in Rotated Sorted Array

Suppose a sorted array is rotated at some pivot unknown to you beforehand.
(i.e., 0 1 2 4 5 6 7 might become 4 5 6 7 0 1 2).

  • You are given a target value to search. If found in the array return its index, otherwise return -1.

  • You may assume no duplicate exists in the array.

  • Example

  • What is Rotated sorted Array?

    • An increasing order array([0,1,2,3,4,5,6,7]) divide into two increasing order parts, first part ([4,5,6,7]), second part ([0,1,2]).
    • Both part is increasing order, but all the elements in first part are bigger than all the elements in second part.
    • Which means we can do binary search in one of the two parts, which depends on the target.

Below is the graph that rotated array will looks like.

Steps and Conditions

  • Divide into have, and compare the target with mid, decide which part we choose to do binary search.
  • Condition 1: (A[mid] == target) => Done!
  • Condition 2: A[start] < A[mid] (Normal binary search in both in red and green part)
    • If target > A[start] but target < mid, then end = mid
    • Otherwise, start = mid
  • Condition 3: A[start] > A[mid] (when mid fall into green line but start still on the red line)
    • If the target on green line, then normal binary search; start = mid.
    • If the target on red line but mid fall to green line: then end = mid and continue divide into half until start and mid on the same line.

What if the Rotated Sorted Array has Duplicates?

  • Sorting based on comparison, the best complexity is $\Theta = nlong n$.
  • “Black box” test.
  • It has to be $\Theta (n)$ Complexity
  • Example: an Rotated Sorted Array with Duplicates
    • [0 0 0 0 0 0 1 0 0]; target = 1
    • if we try complete the search for 1 within $\Theta (log n)$ complexity. It’s impossible.
    • Because only after A[i] travels all the elements in the array, it can show whether the array contains 1.

Merge Sorted Array

Given two sorted integer arrays A and B, merge B into A as one sorted array.

You may assume that A has enough space (size that is greater or equal to m + n) to hold additional elements from B. The number of elements initialized in A and B are m and n respectively.

Median of Two Sorted Arrays

There are two sorted arrays A and B of size m and n respectively. Find the median of the two sorted arrays.

Challenge is Complexity $\Theta (log (k)), k = n+m$

Thoughts

  • Find kth element in two sorted array.
  • If we want to achieve we must make every step divide into half. $k$..$\frac{k}{2}$..$\frac{k}{4}$..$log k$
  • The question become Delete preceding $\frac{k}{2}$ number of elements from both array A and B(total number of elements is $k$). In $\theta (1)$
  • With In $\theta (1)$ complexity, only thing we can do is to change the pointer rather than move elements in array.
  • Either A or B delete the preceding $\frac{k}{2}$ number of element.
  • Then we need to compare $A[\frac{k}{2}]$ and $B[\frac{k}{2}]$ to decide delete from A or B.

Explanation

  • The problem change to find the kth element from two array(from ‘merged’ array)
  • let k = $\frac{(A.length + B.length)}{2}$; which is the position of median in merge array
  • Assume both array A and B have more than $\frac{k}{2}$ elements.
  • We want to Compare A[$\frac{k}{2} - 1$] with B[$\frac{k}{2} - 1$], these two element represent the $\frac{k}{2}$th element in A and B.
  • Total have 3 condtion
    • If A[$\frac{k}{2} - 1$] < B[$\frac{k}{2} - 1$], means all the elements is in between A[0]…A[$\frac{k}{2} - 1$] are in preceding k elements of merged array C.
    • Also A[$\frac{k}{2} - 1$] is impossible bigger than the kth element in array C. So that we can ignore $\frac{k}{2}$ elements from array A.
    • Same conclusion with A[$\frac{k}{2} - 1$] > B[$\frac{k}{2} - 1$].
    • If A[$\frac{k}{2} - 1$] = B[$\frac{k}{2} - 1$]. Means we have find the kth element. Let this value equal to m; since both A and B have $\frac{k}{2}$ elements smaller than m. (2 * ($\frac{k}{2}$)) So that m is the median.
  • Thus we have 3 return condition
    • If A or B is empty, then return B[k-1] or A[k-1]
    • If K = 1, then return the smaller value from A[start] and B[start]
    • If A[$\frac{k}{2} - 1$] = B[$\frac{k}{2} - 1$]. Return either one.

Recover Rotated Sorted Array

Method: Three Step reverse

  • Find the gap of the increasing array. [4 5] [1 2 3]
  • [5 4] [3 2 1] reverse two increasing array
  • [1 2 3 4 5] reverse the whole array

reverse O(1)space; O(n)time

for (int start = 0; end = n - 1; start < end; start++; end –){
swap(A[start] , A[end]);
}
